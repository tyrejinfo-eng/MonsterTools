MonsterTools Intergration

User uses Existing Vs Code GitHub Copilot interface but can now now use it offline and with reliable results.
Vs Code GitHub Copilot interface - copilot proxy tool - Monster Tools - LM Studio Local Model 

This is done so the user can work offline(from both GitHub copilot and LM Studio) and achieve frontier model results with any local LLms as the model is not relied upon as heavily and deterministic workers perform the tasks.
IE/EG: user prompts in VS Code GitHub Copilot to Fix, Implement or create a new engine module or pipeline or implement new code the LLM now becomes a orchestrator or coordinator.
